Dataset: IMDB text reviews with positive/negative sentiment

I chose this data set because I wanted to run a UCCA parser over the text to get a semantic annotation of the text data that wouldve been learned from a corpus of English Treebank reviews from the UCCA resources. I wanted to see what it would be like to try feeding these semantic features into correlative models to see if they produce different results from normal methods that use a purely correlative model and non-semantic text representations like a bag of words model.

This data was produced for a paper published in 2011 and there is not any other information readily available about how the data was collected

http://ai.stanford.edu/~amaas/data/sentiment/

Research question, algorithm choice, initial challenges¶

~~Research question: How does the performance of classifiers for positive/negative sentiment analysis on reviews change when the representations of text is changed to add more features for each particular word. How would a hybrid model perform when additional features from semantic annotations are added into the bag of words model.

I would predict that a hybrid model with more features would add accuracy into the classifier, but if it performs on the same level as models that only use a bag of words for features, then that would mean that the information from semantic annotations is not compatible with simple correlative methods.

Research question: How does the performance of classifiers for positive/negative sentiment analysis on reviews change when the representations of text is changed to use/ignore stop words.

An important difference between semantic annotations of text and bag of word models is that stop words are very important for showing the grammatical structure of text, but they are typically removed in bag of words models. So another interesting question is how do models perform with and without stop words. Perhaps neural nets or n-grams are able to capture some of this grammatical structure by making connections between stop words and other words, so I would expect these methods to perform better than the simple bag of words model

Using a neural net or a naive bayes classifier is well suited for this problem because both of those methods are able to accept large numbers of features for NLP or make associations using stop words

Initial challenges: The reason why I was not able to try the UCCA annotations is because I was having difficulties with getting my C++ devtools setup for the parser package. I was also running out of memory on Jupyter notebook, so I figured that adding in more features would've just made that worse.

To get around some of the memory problems, I decided to just work with a small fraction of the 50,000 rows of text data. I also set the minimum frequency of terms to 5 for the counter, so this is also helping to limit the number of features

Overall results¶

In all of the tests that I performed, the differences in accuracy were usually very small, so it seems like the choice to add or remove stop words might only account for a small difference in results. Stop words seem to account for 1/10 of the words at most, but might only improve accuracy by 1%. When comparing between Naive Bayes and Neural nets, it seems like they result in the similar accuracy scores but Naive Bayes runs much faster. So we should prefer the naive bayes model for the speed.

I ran cross validation on some of the tests, so I think I am confident that the choice of stop words does not change the accuracy of naive bayes but does result in a 1% difference in neural nets where including more features from stop words results in higher accuracy.

Potential harms and benefits¶

I think that representing sentiment for the entire content of a sentence is an oversimplification of speech, so it will not be able to accurately categorize speech from different cultural situations or social norms. For example, in some polite societies saying 'pretty good' is actually the polite way of expressing negative sentiment towards a subject but expressing a positive sentiment towards their audience. So this algorithm using stop words in those ambiguous phrases might not be able to analyze polite speech and could disadvantage people who speak differently.

Another problem with grammatical representations of language is that there is a normative component where the model learns to expect the dominant perspective based on the data. For example, the model might not be able to properly classify text written in African-American Vernacular English based on the different grammatical structure or usage of stop words. So this would disadvantage margninalized groups who use English differently in general, including people who use English as a secondary language.

An advantage of semantic representations in text is that models will be able to handle different contexts in more relevant ways. To augment the models that just find associations between words, a semantic representations can give the model a context independent representation of some words and this could help applications like chat bots give more intelligible responses to unexpected dialogue.

The other advantage of semantic representations is that it gives more interpretability to the model for explaining human speech. Stop words can be used to help represent linguistic rules for implicature and help distinguish between speaker meaning and sentence meaning. This can allow models to extract specific facts from text in ways that are more easily understood.

Future research questions¶

I think the most important difference between the Naive bayes and the Neural nets is that the neural nets saw some increase in accuracy from using the stop words. Even though neural nets are not really interpretable, I think that in some of the hidden layers there are nodes that are learning about grammitical relationships in some ways. To test how this is happening, I would want to think about designing a neural net with some specifications about hidden layers that are meant to encourage finding grammatical relationships. Like if there was a hidden layer that checked if the words as features were used in the same sentence, then that could indicate that a grammatical relationship might be learned instead of a general word occurence relationship.

I am still unsure about how I can actually apply the UCCA data to a neural net implementation, because each word in the annotation will have a data about where in the text it occurs and information about parts of speech tags that cannot be included in a bag of words model, so a new representation might require a much larger amount of features, possibly 15 times bigger to represent all possible combinations of words with parts of speech and possibly 100s of times bigger to represent the location of words in text. Testing this could help us know if our models are getting things correct for the right reasons.

I would also be interested in testing more with bigram or n gram representations, but I was limited by some memory problems. If I had the computational resources then I would've liked to try the models with the full 50k rows of data instead of just 3500.
